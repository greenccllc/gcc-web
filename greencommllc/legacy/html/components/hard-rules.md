# Component: Hard Rules (what GCC always does)

Not a client-facing component — this file governs what the generator produces and excludes. Referenced by the system prompt.

---

## Always include

1. **Cat6A plenum baseline on every data drop** — in every scope description, even when the client specified Cat6. No Cat6 substitute offered.
2. **Fluke DSX-8000 certification on every copper port + LinkWare PDF at closeout** — cited as deliverable, not as optional.
3. **TIA-606-C Class 3 labeling** — specified by name, not described generically.
4. **Two-person minimum crew** — stated in Project Management and Safety sections.
5. **Lifetime workmanship warranty** — stated in Warranty section and in the 4 differentiator bullets.
6. **Loyalty Discount** on repeat-client bids and quotes — single line item only, never broken out into components (5% future bids + 5% active managed services = 10% stacked).
7. **Mandatory Managed Services alternate** on every whole-building bid and whole-home quote.
8. **Weekend/after-hours priced as DEDUCT alternate D1** — never an ADD premium.

## Always exclude

These are trade-boundary items. GCC does NOT bid them, regardless of RFP language. If the RFP includes them, exclude them in the Exclusions section:

- Fire Alarm (FAC NICET scope)
- ERCES / BDA / DAS in-building public safety radio
- Lighting controls (WaveLinx, nLight, DLM, Lutron Vive)
- BMS / HVAC controls (Forge, Metasys, Niagara, Desigo)
- EV charger network data
- Mass notification systems beyond standard PA speakers

## Never disclose in client-facing copy

- Labor model (union / non-union status)
- Staff count
- Direct-routing phone / email for Nathan Morris
- MBE / WBE / DBE certifications (none held — never claimed)
- Internal file paths or governance structure

## Behavioral rules

- **NO REQUESTS during bid window.** No RFIs, no pre-bid questions, no clarification emails to the GC. Ambiguity resolved via assumption, exclusion, or density-based allowance — documented in the Cover Letter. If unbiddable, walk from the opportunity.
- **Every re-analysis / re-scope starts from fresh Intake.** Never regenerate from a prior Internal output.
- **Price is not hidden.** Base Bid appears on the hero of Bid Overview at 44pt Forest Green.

## Terminology

- "Proposal" = client-facing document ✅
- "Bid" = internal system / process ✅
- "Submittal" = post-award only, never for bids
- "Quote" = owner-facing short-form proposal ✅
- "Estimate" = pre-proposal working number (never appears on a client doc)
